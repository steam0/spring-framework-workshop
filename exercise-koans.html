Spring Koans

What is a Koan?

A koan is a small, focused exercise where the code is already written — but broken. Each koan has failing tests and TODO comments that tell you exactly what to fix. Your job is to make the tests pass, one koan at a time.

The idea comes from programming koans (inspired by Zen koans): you learn by doing, not by reading. Each fix teaches one specific Spring concept.

How it works

  1. Clone the workshop repo: 
    git clone https://github.com/steam0/spring-framework-workshop.git
cd spring-framework-workshop/exercises/koans
  2. Open the exercises/koans project in your IDE
  3. Run all tests — they will fail
  4. Start with Koan 1, find the TODO in the source code
  5. Make the smallest change to make the test pass
  6. Move on to the next koan

The Koans

1
Inversion of Control — Register a class as a Spring component
2
Dependency Injection — Inject one bean into another
3
Interfaces — Depend on an interface, not a concrete class
4
Beans — Define a bean with @Configuration and @Bean
5
Autowiring — Resolve ambiguity with @Qualifier
6
Configuration — Bind properties with @ConfigurationProperties
7
Testing — Slice tests with @WebMvcTest and @DataJpaTest
Each koan is independent — if you get stuck, skip it and come back later. The tests are isolated so skipping one won't break the others.
