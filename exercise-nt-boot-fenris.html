nt-boot-fenris

Build your own nt-boot module. nt-boot-fenris should auto-configure a database connection for any application that includes it — with per-environment connection pool settings.

Setup

  1. Clone the nt-boot-4 repo: 
    git clone https://bitbucket.norsk-tipping.no/scm/tr/nt-boot-4.git
  2. Create a branch for your work: 
    cd nt-boot-4
git checkout -b feature/nt-boot-fenris
  3. Look at how existing modules (e.g. nt-boot-core, nt-boot-auth) are structured — your module will follow the same pattern

The scenario

Every team at NT needs a database. Today, each team configures their own DataSource, connection pool, and timeouts. The settings are copy-pasted between projects and often wrong for production. Your job: make a module that handles this automatically.

Requirements

  1. Create a Maven module called nt-boot-fenris with groupId no.norsktipping.boot
  2. Define a @ConfigurationProperties class bound to nt.fenris with:
    • url — JDBC connection URL
    • username / password
    • pool.max-size — maximum connections (default: 10)
    • pool.min-idle — minimum idle connections (default: 2)
    • pool.connection-timeout — how long to wait for a connection (default: 3s). A short timeout is intentional — in a Kubernetes environment it's better to fail fast and let the caller retry or the pod restart, rather than blocking threads waiting for a connection that may never come.
  3. Create an auto-configuration class that:
    • Configures a HikariDataSource using the properties above
    • Only activates when nt.fenris.url is set
    • Backs off if the application already defines its own DataSource
  4. Register your auto-configuration so Spring Boot discovers it

Per-environment configuration

Once the module works, the consuming application just sets properties per profile:

nt.fenris:
  url: jdbc:h2:mem:devdb
  username: sa
  password: 
  pool:
    max-size: 5
    min-idle: 1
    connection-timeout: 10s
nt.fenris:
  url: jdbc:postgresql://db.prod.internal:5432/fenris
  username: ${DB_USER}
  password: ${DB_PASSWORD}
  pool:
    max-size: 30
    min-idle: 5
    connection-timeout: 5s

Hints

Auto-configuration registration
Spring Boot discovers auto-configurations via META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports — one fully-qualified class name per line.
Conditional activation
Use @ConditionalOnProperty(name = "nt.fenris.url") so the module does nothing unless configured.
Backing off
Use @ConditionalOnMissingBean(DataSource::class) so apps can override your DataSource if they need to.
HikariCP
Spring Boot includes HikariCP by default. Create a HikariDataSource and set jdbcUrl, maximumPoolSize, minimumIdle, and connectionTimeout.
Duration properties
Use java.time.Duration for the timeout — Spring Boot automatically converts 10s, 500ms, etc.
This is exactly how the real nt-boot modules work. A single dependency in your pom.xml, a few lines of YAML per environment, and the module handles the rest.
