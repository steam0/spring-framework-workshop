nt-boot-unleash

Feature flag management with Unleash. Add the dependency and configure the server — you get an Unleash bean ready to inject.

<dependency>
    <groupId>no.norsktipping.boot</groupId>
    <artifactId>nt-boot-unleash</artifactId>
</dependency>
nt:
  unleash:
    api-url: "https://unleash.norsk-tipping.no/api"
    api-token: "${UNLEASH_API_TOKEN}"
    instance-id: "${HOSTNAME}"
@Service
class CheckoutService(
    private val unleash: Unleash
) {
    fun checkout() {
        if (unleash.isEnabled("new-checkout-flow")) {
            // new implementation
        } else {
            // old implementation
        }
    }
}
@Service
public class CheckoutService {

    private final Unleash unleash;

    public CheckoutService(Unleash unleash) {
        this.unleash = unleash;
    }

    public void checkout() {
        if (unleash.isEnabled("new-checkout-flow")) {
            // new implementation
        } else {
            // old implementation
        }
    }
}

How It Works

  • Polls the Unleash server every 15 seconds and caches toggle state locally
  • isEnabled() evaluates against the local cache — no network call
  • If the server is down, the SDK uses the last known configuration
  • On cold start with no server available, all toggles return false

Gradual Rollout

Unleash supports gradual rollout strategies using Murmurhash3 of the toggle name combined with a user context. All evaluation happens locally — no user data is sent to the Unleash server.

val context = UnleashContext.builder()
    .userId(user.personId)
    .sessionId(sessionId)
    .build()

if (unleash.isEnabled("new-checkout-flow", context)) {
    // user is in the rollout group
}
UnleashContext context = UnleashContext.builder()
    .userId(user.getPersonId())
    .sessionId(sessionId)
    .build();

if (unleash.isEnabled("new-checkout-flow", context)) {
    // user is in the rollout group
}
